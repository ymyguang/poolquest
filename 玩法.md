# PoolQuest 玩法与规则 V1.0

> 本文件是 PoolQuest 的完整玩法定义。所有数值、公式、约束均为最终设计。
> 旧版 V0.1 示例（Dragon Pool 固定模板）已废弃。副本由 LLM 动态生成。

---

# 1. PoolQuest 是什么？

**PoolQuest** 是一个基于 Uniswap V4 Hook 的链上副本游戏平台。

- 创作者用 Prompt 描述副本主题和意图
- LLM 生成完整的隐藏路径、世界观、谜语反馈、提示梯度
- 玩家用真实 AMM 操作（Swap、LP、Donate 等）探索隐藏路径
- Hook 合约在链上记录操作、进度、通关结果
- 排行榜按 Quest Score 排名，通关者瓜分奖池

一句话：

> **创作者出题，LLM 造谜，Hook 记录，玩家用交易闯关。**

# 2. 核心角色

## 2.1 Creator 创作者

```
职责：
1. 描述副本主题和意图（Prompt）
2. 选择难度
3. 发布副本

MVP 阶段创建免费，不需要支付费用。
发布前系统自动验证副本可通关性。
```

## 2.2 Player 玩家

```
职责：
1. 选择副本
2. 支付 Entry Fee 进场
3. 通过 AMM 操作探索隐藏路径
4. 冲击排行榜

玩家可用操作：
  Swap（买入/卖出 AgentToken）
  Add LP / Remove LP
  Donate（向池子捐赠 QUSD）
  Hold（等待条件满足）
  Ask Agent（付费追问提示）
```

## 2.3 Agent 叙事者

```
职责：
1. 讲故事，维持世界观
2. 给免费谜语反馈（不直接确认对错）
3. 卖付费提示（从模糊到接近答案）

Agent 不决定胜负。
Rule Engine + Hook 决定游戏状态。
```

## 2.4 Hook 链上记录层

```
职责：
1. 记录玩家 AMM 操作
2. 记录经 Rule Engine 确认的进度
3. 记录提示购买
4. 记录通关结果
5. 发出可索引事件

Hook 不存储完整隐藏路径。
完整规则由后端存储，Hook 只记录 ruleHash / solutionHash。
```

# 3. 副本资产模型

每个副本绑定一个真实的 Uniswap V4 交易对：

```
AgentToken / QUSD
```

其中：
- QUSD = PoolQuest 平台币，全场通用消耗资产
- AgentToken = 副本专属资产，由 LLM 根据 Agent 身份命名

例如 LLM 可能生成：
```
Dragon Agent → DRAGON / QUSD
Witch Agent  → WITCH / QUSD
Monk Agent   → MONK / QUSD
```

每个已发布 Agent 对应：
```
1. 一个 AgentToken ERC20
2. 一个 AgentToken/QUSD 的 Uniswap V4 Pool
3. 一个绑定该 Pool 的 Hook
4. 一套后端存储的隐藏规则 + 链上 ruleHash/solutionHash
```

# 4. 单局 Run 参数

| 项目 | 数值 |
|---|---|
| 初始挑战资金 | 100 QUSD |
| 时间限制 | 10 分钟 |
| 标准步骤数 | 5 步（LLM 可选 4-6 步） |
| Entry Fee | 5 QUSD |
| AMM LP Fee | 0.30% |
| Hook Fee | 0.05% |
| 通关条件 | 完成全部隐藏步骤 + 最终净值 ≥ 85 QUSD |
| 重复挑战 | 同一副本可重复挑战 |
| 排行榜 | 每个钱包只取最高分 Run |

# 5. 经济系统

## 5.1 Entry Fee 分配

玩家进场支付 5 QUSD：

| 去向 | 比例 | 金额 |
|---|---|---|
| 副本奖池 | 60% | 3 QUSD |
| 平台 | 20% | 1 QUSD |
| 创作者收益 | 10% | 0.5 QUSD |
| 保护基金 | 10% | 0.5 QUSD |

## 5.2 Hook Fee

玩家在副本中 Swap 时收取 0.05% Hook Fee。

MVP 阶段：Hook Fee 100% 归平台（简化实现）。

## 5.3 Hint Fee

玩家付费追问提示，费用以 QUSD 支付。

MVP 阶段：Hint Fee 100% 进入副本奖池。

> 用提示 = 增加奖池 = 对不用提示的高手有利。
> 创作者/平台不从提示中获利，没有诱导消费的动机。

## 5.4 创作者收益

MVP 阶段创建免费，创作者收益来源：
- Entry Fee 分成：0.5 QUSD/人
- 主要激励是社区声望和副本参与人数

## 5.5 冷启动

首期平台注入 200 QUSD 种子奖池，解决"没有玩家就没有奖池"的冷启动问题。

## 5.6 玩家盈亏分析

```
100 人参与时，奖池 ≈ 200（种子）+ 300（Entry Fee）= 500 QUSD

第 1 名：500 × 30% = 150 QUSD（支出 5.5，净赚 144.5）
前 10 名：500 × 25% / 7 ≈ 17.8 QUSD（支出 5.5，净赚 12.3）
前 50 名：500 × 20% / 40 ≈ 2.5 QUSD（支出 5.5，净亏 3.0）
未通关：净亏 5.5 QUSD（可预期的娱乐成本）
```

玩家进场前就知道最多亏 5.5 QUSD，这是可接受的娱乐消费。

# 6. 隐藏路径生成

## 6.1 操作库

LLM 必须从以下操作中组合路径：

```
donate            → 向池子捐赠 QUSD
add_liquidity     → 添加流动性
remove_liquidity  → 移除流动性
swap_buy          → 买入 AgentToken
swap_sell         → 卖出 AgentToken
hold_duration     → 等待 N 秒（N ∈ [60, 300]）
hold_price        → 等池价达到某个方向后操作
combo             → 两个操作按顺序执行（如 Buy → LP）
```

## 6.2 路径约束

```
1. 路径长度：标准 5 步（可选 4-6 步，由难度决定）
2. 每步必须是可验证的（链上事件或时间戳）
3. 路径必须能在 10 分钟内完成
4. 路径必须能在 100 QUSD 初始资金下完成
5. 最终净值必须 ≥ 85 QUSD（模拟验证）
6. 不能有不可避免的惩罚
7. 第一步和最后一步不能相同
8. 相邻两步不能是互斥操作（如 Buy 后立刻 Sell）
```

## 6.3 路径示例（LLM 生成，非固定模板）

示例 A：
```
Donate → Buy → Add LP → Hold 120s → Sell
阶段名：拜访 → 钥匙 → 锁链 → 沉默 → 突破
```

示例 B：
```
Add LP → Donate → Buy → Hold 180s → Sell
阶段名：扎根 → 献祭 → 收获 → 等待 → 离开
```

示例 C：
```
Buy → Add LP → Hold 60s → Donate → Sell
阶段名：点燃 → 铺路 → 屏息 → 祈祷 → 穿越
```

每次生成都是不同的组合和叙事，同一个主题不会出现相同的路径。

# 7. 开局预言（Opening Prophecy）

每个副本在开始时提供一条开局预言。

## 7.1 约束

```
1. 必须是谜语人风格，不能直接说操作名
2. 只暗示前 1-2 步的方向，不是全部 5 步
3. 必须和世界观一致
4. 玩家读完后应该知道"要做什么类型的事"，但不知道"具体做什么"
5. 必须和真实 Hook 规则一致
```

## 7.2 示例（LLM 生成）

路径：`Donate → Buy → Add LP → Hold 120s → Sell`

```
"古老的巨龙只接受诚心的拜访者。
 先证明你的诚意，再谈金币的事。"
```

暗示：先 Donate，然后和金币有关。不暗示：LP、Hold、Sell。

# 8. 免费反馈规则

玩家每执行一步操作后，获得一次免费谜语反馈。

## 8.1 触发条件

```
Swap / Add LP / Remove LP / Donate / Hold 到达关键时间点
触发状态变化 / 触发惩罚 / 触发祝福
```

## 8.2 约束

```
1. 必须是谜语人风格，不能直接确认操作正确/错误
2. 使用世界观语言，不用操作名
3. 正确操作：暗示方向对了，但不确认
4. 错误操作：暗示方向不对，但不指出正确方向
5. 危险操作：警告但不说为什么
```

## 8.3 示例（LLM 生成）

路径第三步是 Buy，玩家执行了 Buy：
```
"有什么东西在暗处动了一下。也许它注意到了你。"
```

路径第三步是 Buy，玩家执行了 Sell：
```
"你转身的时候，身后有什么东西安静下来了。那不是好的安静。"
```

路径第三步是 Buy，玩家执行了 Add LP：
```
"有什么东西像盔甲，但它还没有披到身上。"
```

> 关键：玩家不能通过单次反馈确定自己是否做对了，需要多次试探累积判断。

# 9. 付费提示规则

玩家可以主动向 Agent 追问提示。

## 9.1 提示等级

| 等级 | 说明 | 内容风格 |
|---|---|---|
| Level 1 | 方向提示 | "你现在应该关注什么" |
| Level 2 | 范围提示 | "下一步操作的类型" |
| Level 3 | 接近提示 | "具体该做什么"（世界观语言） |

## 9.2 提示费用公式

```
Hint Fee = Base Fee × Repeat Multiplier × Progress Multiplier
```

### 基础价格

| 提示等级 | Base Hint Fee |
|---|---|
| Level 1 | 0.2 QUSD |
| Level 2 | 0.6 QUSD |
| Level 3 | 1.5 QUSD |

### 追问次数倍率（同一 Run 内）

| 第几次追问 | Repeat Multiplier |
|---|---|
| 第 1 次 | ×1 |
| 第 2 次 | ×1.5 |
| 第 3 次 | ×2.5 |
| 第 4 次 | ×4 |
| 第 5 次及以后 | ×6 |

### 进度倍率

| 当前进度 | Progress Multiplier |
|---|---|
| Step 0/5 | ×1 |
| Step 1/5 | ×1 |
| Step 2/5 | ×1.5 |
| Step 3/5 | ×2 |
| Step 4/5 | ×3 |
| 最后一步前 | ×5 |

> 越靠近最后，状态空间越小，提示越接近答案，费用越高。

## 9.3 提示费用示例

玩家在 Step 3/5，第一次追问 Level 1：
```
Hint Fee = 0.2 × 1 × 2 = 0.4 QUSD
```

玩家在 Step 3/5，第二次追问 Level 2：
```
Hint Fee = 0.6 × 1.5 × 2 = 1.8 QUSD
```

玩家在 Step 4/5，第三次追问 Level 3：
```
Hint Fee = 1.5 × 2.5 × 3 = 11.25 QUSD
```

## 9.4 提示内容约束

Level 1（方向）：
```
"火焰需要时间，金币需要耐心"
→ 暗示 Hold，但不说具体等多久
```

Level 2（范围）：
```
"让时间站在你这边"
→ 更明显暗示 Hold/等待，但还是谜语
```

Level 3（接近）：
```
"安静地坐在龙穴里，等火焰低头"
→ 几乎直接说 Hold，但仍是世界观语言
```

## 9.5 提示扣分

| 提示等级 | 基础扣分 |
|---|---|
| Level 1 | -50 分 |
| Level 2 | -150 分 |
| Level 3 | -400 分 |

实际扣分公式：
```
Hint Penalty = Base Penalty × Repeat Multiplier × Progress Multiplier
```

> 高手不用提示冲榜；新手靠提示通关但分数较低。这是有意设计。

# 10. Quest Score 计分

排行榜按 Quest Score 排名。

```
Quest Score =
  Completion Score
  + Net Worth Score
  + Time Score
  + Efficiency Score
  + LP Contribution Score
  - Hint Penalty
  - Curse Penalty
```

## 10.1 Completion Score

| 结果 | 分数 |
|---|---|
| 完成通关 | +1000 |
| 未通关 | 0 |

## 10.2 Net Worth Score

```
Net Worth Score = (Final Net Worth - 100) × 20
限制：最低 -500，最高 +1000
```

| 最终净值 | 分数 |
|---|---|
| 85 QUSD | -300 |
| 100 QUSD | 0 |
| 110 QUSD | +200 |
| 125 QUSD | +500 |
| 150 QUSD | +1000 |

## 10.3 Time Score

```
Time Score = 300 × (1 - 用时秒数 / 600)
限制：最低 0，最高 300
```

| 通关时间 | 分数 |
|---|---|
| 2 分钟 | 200 |
| 5 分钟 | 150 |
| 8 分钟 | 60 |
| 10 分钟 | 0 |

## 10.4 Efficiency Score

| 操作次数 | 分数 |
|---|---|
| 5 步完成 | +300 |
| 6 步完成 | +250 |
| 7 步完成 | +200 |
| 8 步完成 | +150 |
| 9 步完成 | +100 |
| 10 步完成 | +50 |
| 10 步以上 | 0 |

## 10.5 LP Contribution Score

| 行为 | 分数 |
|---|---|
| Add LP 并保持 ≥ 2 分钟 | +100 |
| Add LP 并保持 ≥ 3 分钟 | +200 |
| Add LP 并保持 ≥ 5 分钟 | +300 |
| 快进快出 LP（< 60 秒） | -150 |

上限：+300 分

## 10.6 Curse Penalty

| 惩罚类型 | 扣分 |
|---|---|
| Minor Curse | -100 |
| Major Curse | -300 |
| Fatal Curse | -700 |

示例：Buy 后 60 秒内 Sell → Quick Sell Curse → -300 分

## 10.7 Score 上限

Quest Score 上限：3600 分（全部满分理论值）

# 11. 奖池分配

## 11.1 奖池来源

```
1. 平台种子注入（首期 200 QUSD）
2. Entry Fee 的 60%（3 QUSD/人）
3. Hint Fee 的 100%
```

## 11.2 排行榜奖励

每个副本排行榜前 50 名瓜分奖池。只有通关玩家有资格。

| 排名 | 奖池比例 |
|---|---|
| 第 1 名 | 30% |
| 第 2 名 | 15% |
| 第 3 名 | 10% |
| 第 4-10 名 | 25%（按分数占比） |
| 第 11-50 名 | 20%（按分数占比） |

## 11.3 无人通关

```
无人通关 → 奖池滚入下一期
```

# 12. 进度显示

玩家看不到真实操作步骤，但能看到角色化进度。

## 12.1 阶段显示

每个副本有 5 个可见阶段名（由 LLM 生成，叙事化）。

例如 LLM 生成：
```
拜访 → 钥匙 → 锁链 → 沉默 → 突破
[ ]    [ ]    [ ]    [ ]    [ ]
```

阶段名和真实操作的对应关系对玩家隐藏。

## 12.2 进度状态符号

| 符号 | 含义 |
|---|---|
| `[ ]` | 尚未触发 |
| `[?]` | 可能相关，但未确认 |
| `[✓]` | 已正确触发 |
| `[!]` | 当前处于危险状态 |
| `[×]` | 已触发惩罚 |
| `[★]` | 已完成通关 |

# 13. 创作者创建流程

## 13.1 创建

MVP 阶段创建免费。

创作者提供：
```
1. Agent 名称
2. 主题描述
3. Agent 人设
4. 难度
5. 提示风格（谜语人、直白、幽默等）
6. 初始奖池金额（可选，最低 0 QUSD）
```

## 13.2 AI 生成

系统调用 LLM 生成：
```
publicProfile: 开局预言、可见阶段名、Agent 人设
privateRule: 隐藏路径、惩罚规则、祝福规则、提示梯度
```

## 13.3 可通关性验证

系统自动模拟验证：
```
1. 路径是否可以完成
2. 需要的最小资金是否 ≤ 100 QUSD
3. 是否存在无法避免的惩罚
4. 10 分钟内是否能完成
5. 最终净值是否 ≥ 85 QUSD
```

验证不通过，不允许发布。

## 13.4 发布

```
1. 部署 AgentToken
2. 部署或配置 Hook
3. 初始化 Uniswap V4 AgentToken/QUSD Pool
4. 注册到 PoolQuestRegistry
5. 存储 ruleHash / solutionHash
6. 显示合约地址和 explorer 链接
```

## 13.5 创作者权限

```
创作者可以查看自己副本的完整隐藏路径
玩家永远不能看到真实路径
```

# 15. 网站极简说明（面向用户）

```
1. 选择一个 Agent 副本
2. 支付 5 QUSD 入场费
3. 使用 100 QUSD 标准资金开始挑战
4. 根据开局预言推理正确路径
5. 通过 Swap、Add LP、Donate 等操作试探
6. 每一步操作后，Agent 会给你一句谜语反馈
7. 如果看不懂，可以付费追问提示
8. 追问越多越贵，越接近通关越贵
9. 使用提示会扣排行榜分
10. 成功通关后获得 Quest Score
11. 排行榜前 50 名瓜分副本奖池
```
