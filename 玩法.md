# PoolQuest 游戏规则 V0.1

## 1. PoolQuest 是什么？

**PoolQuest** 是一个基于 Uniswap V4 Hook 的链上副本游戏。

每个副本都是一个真实的 AMM Pool。玩家不是单纯交易，而是通过 **Swap、Add LP、Remove LP、Donate、Hold** 等链上操作探索隐藏规则、触发状态、完成任务、冲击排行榜并获得奖励。

一句话：

> **创作者设计谜题，Agent 说谜语，Hook 执行规则，玩家用交易闯关。**
> 

---

# 2. 核心角色

## 2.1 创作者 Creator

创作者可以用 Prompt 创建一个 Agent 副本。

创作者负责：

```
1. 设定 Agent 角色
2. 选择副本主题
3. 注入初始奖池
4. 提交隐藏通关路径
5. 获得手续费和提示费分成
```

例如：

```
创建一个巨龙副本。
巨龙守着宝藏。
玩家需要通过献祭、披甲、取金币、等待火焰平息，最后带走宝藏。
提示必须是谜语，不能直接告诉玩家答案。
```

---

## 2.2 挑战者 Player

挑战者进入副本，用自己的游戏资金完成挑战。

玩家可以：

```
Swap：买入或卖出副本 Token
Add LP：提供流动性
Remove LP：撤出流动性
Donate：献祭给副本
Hold：等待一段时间
Ask Agent：向 Agent 付费追问提示
```

---

## 2.3 Agent

Agent 是副本主持人。

Agent 不直接决定链上结果。

链上结果由 Hook 判断。

Agent 负责：

```
1. 讲故事
2. 给免费模糊反馈
3. 给付费谜语提示
4. 根据玩家状态进行个性化回应
```

---

## 2.4 Hook

Hook 是副本的链上可验证执行与记录层。

MVP 中，完整隐藏规则由后台保存和评估；Hook 负责在 X Layer 上记录关键 AMM 操作、进度、惩罚、提示、通关结果以及规则承诺哈希。

Hook 负责：

```
1. 记录玩家关键链上操作
2. 记录或接收经授权规则引擎确认的玩家进度
3. 触发并记录祝福、诅咒、护盾、宝藏等状态
4. 记录 Quest Score 相关事件
5. 结算或证明副本结果
6. 保存 ruleHash / solutionHash 等规则承诺
```

# 3. 副本资产规则

每个 Agent 副本都会创建或绑定一个真实的 Agent 角色主币。

这个 Agent 角色主币就是该副本池子的主资产，名称和符号来自 Agent 本身。

例如创建 Dragon Agent 时：

```
Agent 名称：Dragon
Agent 角色主币：DRAGON
真实池子：DRAGON / QUSD
```

每个副本都绑定一个真实的 Uniswap V4 交易对：

```
Agent Token / QUSD
```

其中：

```
QUSD = PoolQuest 平台币 / 全场通用消耗资产 / 奖池资产
Agent Token = Agent 角色主币 / 副本专属资产
```

例如：

```
Dragon Pool：DRAGON / QUSD
Witch Pool ：WITCH  / QUSD
Monk Pool  ：MONK   / QUSD
Devil Pool ：DEVIL  / QUSD
```

玩家进入 Dragon Pool，就会围绕 `DRAGON / QUSD` 进行 Swap、LP、Donate 等操作。

因此，Agent 不是单纯的前端角色卡，也不是只存在数据库里的 NPC。

每个已发布 Agent 至少对应：

```
1. 一个 Agent Token
2. 一个 AgentToken / QUSD 的真实 Uniswap V4 Pool
3. 一个绑定该 Pool 的 Hook
4. 一套后台保存的隐藏规则和链上 ruleHash / solutionHash
```

# 4. 每局挑战的基础参数

每次挑战称为一次 **Run**。

默认参数如下：

| 项目 | 数值 |
| --- | --- |
| 单局初始挑战资金 | 100 QUSD |
| 单局时间限制 | 15 分钟 |
| 标准副本步骤数 | 5 步 |
| 入场费 Entry Fee | 1 QUSD |
| 默认 AMM LP Fee | 0.30% |
| 默认 Hook Fee | 0.05% |
| 单个钱包可重复挑战 | 可以 |
| 排行榜计分 | 每个钱包只取最高分 Run |

玩家进入副本时，需要支付：

```
1 QUSD Entry Fee
```

同时本局挑战资金标准化为：

```
100 QUSD
```

# 5. 通关条件

一个标准副本默认有 5 个隐藏步骤。

玩家需要在 15 分钟内完成隐藏路径，并满足最终净值要求。

## 默认通关条件

```
1. 完成 5 个隐藏步骤
2. 在 15 分钟内完成
3. 最终净值 ≥ 95 QUSD
```

最终净值计算方式：

```
最终净值 =
QUSD 余额
+ Agent Token 按当前池价折算的 QUSD 价值
+ LP 仓位折算价值
+ 可领取的副本奖励
```

注意：

```
Entry Fee 和 Hint Fee 不计入最终净值。
它们是挑战成本。
```

# 6. 副本路径与进度显示

每个副本都有一条隐藏路径。

玩家看不到真实操作步骤，但能看到角色化进度。

例如 Dragon Pool 的真实路径是：

```
Donate → Add LP → Buy → Hold → Sell
```

但玩家看到的是：

```
Dragon Trial Path

烟雾        龙鳞        金币        火焰        宝藏
[ ] ────── [ ] ────── [ ] ────── [ ] ────── [ ]
```

对应关系：

| 显示给玩家 | 真实操作 |
| --- | --- |
| 烟雾升起 | Donate |
| 龙鳞披身 | Add LP |
| 金币入爪 | Buy |
| 火焰低头 | Hold |
| 宝藏开门 | Sell |

玩家每完成一步，进度会更新。

---

## 进度状态符号

| 符号 | 含义 |
| --- | --- |
| `[ ]` | 尚未触发 |
| `[✓]` | 已正确触发 |
| `[?]` | 可能相关，但未确认 |
| `[!]` | 当前处于危险状态 |
| `[×]` | 已触发惩罚 |
| `[★]` | 已完成通关 |

示例：

```
Dragon Trial Path

烟雾        龙鳞        金币        火焰        宝藏
[✓] ────── [✓] ────── [✓] ────── [!] ────── [ ]

当前状态：
你已经拿起金币，但火焰还没有低头。
```

这表示玩家已经完成前 3 步，但现在不能贸然退出。

# 7. 开局提示规则

每个副本必须在开始时提供一条 **Opening Prophecy / 开局预言**。

开局提示必须满足：

```
1. 暗示正确路径
2. 符合 Agent 世界观
3. 不能直接说出具体操作
4. 必须和真实 Hook 规则一致
```

例如 Dragon Pool：

```
巨龙盘踞在金山之上。

“献上第一缕烟，
披上沉默的鳞，
握住金子后别急着飞走。
等火焰低头，门会自己打开。”
```

这句话暗示：

```
Donate → Add LP → Buy → Hold → Sell
```

但不会直接写明操作。

# 8. 免费反馈规则

玩家每执行一步操作后，都会获得一次 **免费模糊反馈**。

免费反馈不收 Token。

触发免费反馈的动作包括：

```
Swap
Add LP
Remove LP
Donate
Hold 到达关键时间点
触发状态变化
触发惩罚
触发祝福
```

免费反馈只告诉玩家：

```
走得对不对
是否接近目标
是否处于危险状态
是否缺少某种前置条件
```

但不会直接告诉下一步该做什么。

---

## 免费反馈示例

玩家 Buy 后：

```
“你拿起了金币，巨龙睁开了一只眼。”
```

玩家 Buy 后立刻 Sell：

```
“金币还烫着，急着离开的人会被火记住。”
```

玩家 Add LP 后：

```
“鳞片覆盖了你的手，火不会立刻认出你。”
```

---

# 9. 付费追问规则

如果玩家看不懂免费反馈，可以主动向 Agent 追问。

付费追问规则：

```
1. 主动追问需要支付 Agent Token
2. 追问越多，费用越高
3. 越接近通关，费用越高
4. 提示越准确，扣除的 Quest Score 越多
```

---

# 10. 提示费用计算公式

提示费用使用以下公式：

```
Hint Fee = Base Hint Fee × Hint Level Multiplier × Repeat Multiplier × Progress Multiplier
```

提示费以 **QUSD** 支付。

QUSD 是平台统一消耗资产。提示费不再换算成 Agent Token。Agent Token 仍用于对应副本池子的 Swap / LP 操作。

---

## 10.1 提示等级基础价格

| 提示等级 | 说明 | Base Hint Fee |
| --- | --- | --- |
| Level 1 | 方向提示 | 0.2 QUSD |
| Level 2 | 较明确提示 | 0.6 QUSD |
| Level 3 | 接近答案提示 | 1.5 QUSD |

---

## 10.2 提示等级倍率

| 提示等级 | 倍率 |
| --- | --- |
| Level 1 | ×1 |
| Level 2 | ×1 |
| Level 3 | ×1 |

基础价格已经区分等级，所以等级倍率默认为 1。

---

## 10.3 追问次数倍率

同一局 Run 内，玩家追问越多，费用越高。

| 本局第几次追问 | Repeat Multiplier |
| --- | --- |
| 第 1 次 | ×1 |
| 第 2 次 | ×1.5 |
| 第 3 次 | ×2.5 |
| 第 4 次 | ×4 |
| 第 5 次及以后 | ×6 |

---

## 10.4 进度倍率

副本默认 5 步。

玩家越接近最后一步，提示越贵。

| 当前确认进度 | Progress Multiplier |
| --- | --- |
| Step 0 / 5 | ×1 |
| Step 1 / 5 | ×1 |
| Step 2 / 5 | ×1.5 |
| Step 3 / 5 | ×2 |
| Step 4 / 5 | ×3 |
| Step 5 前最后状态 | ×5 |

原因：

> 越靠近最后，状态空间越小，提示越接近答案，所以费用越高。
> 

---

## 10.5 提示费用示例

假设玩家在 Dragon Pool 中，当前进度为 Step 3 / 5。

玩家第一次追问 Level 1 提示：

```
Hint Fee = 0.2 × 1 × 1 × 2 = 0.4 QUSD
```

玩家第二次追问 Level 2 提示：

```
Hint Fee = 0.6 × 1 × 1.5 × 2 = 1.8 QUSD
```

玩家第三次追问 Level 3 提示：

```
Hint Fee = 1.5 × 1 × 2.5 × 2 = 7.5 QUSD
```

如果玩家已经到 Step 4 / 5，再问 Level 3：

```
Hint Fee = 1.5 × 1 × 2.5 × 3 = 11.25 QUSD
```

如果只差最后一步，Progress Multiplier = ×5，费用会更高。

# 11. 提示扣分规则

付费提示不仅需要 Token，也会扣排行榜分。

| 提示等级 | 基础扣分 |
| --- | --- |
| Level 1 | -50 分 |
| Level 2 | -150 分 |
| Level 3 | -400 分 |

实际扣分公式：

```
Hint Penalty = Base Penalty × Repeat Multiplier × Progress Multiplier
```

例如：

玩家在 Step 4 / 5，第 3 次追问 Level 2：

```
Hint Penalty = 150 × 2.5 × 3 = 1125 分
```

这意味着：

```
普通玩家可以靠提示通关；
高手玩家想冲榜，就要少问甚至不问。
```

---

# 12. Quest Score 计分规则

排行榜按照 Quest Score 排名。

Quest Score 计算公式：

```
Quest Score =
Completion Score
+ Treasure Score
+ Net Worth Score
+ Time Score
+ Efficiency Score
+ LP Contribution Score
- Hint Penalty
- Curse Penalty
- Anti-Wash Penalty
```

---

## 12.1 Completion Score 通关分

| 结果 | 分数 |
| --- | --- |
| 完成主线通关 | +1000 |
| 未通关 | +0 |

---

## 12.2 Treasure Score 隐藏宝藏分

| 结果 | 分数 |
| --- | --- |
| 触发隐藏 Treasure | +500 |
| 未触发 | +0 |

---

## 12.3 Net Worth Score 净值分

初始挑战资金为：

```
100 QUSD
```

最终净值分公式：

```
Net Worth Score = (Final Net Worth - 100) × 20
```

限制：

```
最低 -500 分
最高 +1000 分
```

示例：

| 最终净值 | 净值分 |
| --- | --- |
| 90 QUSD | -200 |
| 100 QUSD | 0 |
| 110 QUSD | +200 |
| 125 QUSD | +500 |
| 150 QUSD | +1000 |

---

## 12.4 Time Score 时间分

单局时间限制为 15 分钟。

通关越快，分越高。

公式：

```
Time Score = 300 × (1 - 用时秒数 / 900)
```

限制：

```
最低 0 分
最高 300 分
```

示例：

| 通关时间 | 时间分 |
| --- | --- |
| 5 分钟 | 200 |
| 10 分钟 | 100 |
| 15 分钟 | 0 |

---

## 12.5 Efficiency Score 效率分

每个标准副本有一条最短参考路径，默认 5 步。

计分规则：

| 操作次数 | 效率分 |
| --- | --- |
| 5 步完成 | +300 |
| 6 步完成 | +250 |
| 7 步完成 | +200 |
| 8 步完成 | +150 |
| 9 步完成 | +100 |
| 10 步完成 | +50 |
| 10 步以上 | +0 |

---

## 12.6 LP Contribution Score

LP 行为可以加分，但必须真实持续。

| 行为 | 分数 |
| --- | --- |
| Add LP 并保持 ≥ 3 分钟 | +150 |
| Add LP 并保持 ≥ 5 分钟 | +250 |
| Add LP 并保持 ≥ 10 分钟 | +350 |
| 快进快出 LP，少于 60 秒撤出 | -200 |

每局 LP Contribution Score 上限：

```
+350 分
```

---

## 12.7 Curse Penalty

触发副本惩罚会扣分。

| 惩罚类型 | 扣分 |
| --- | --- |
| Minor Curse | -100 |
| Major Curse | -300 |
| Fatal Curse | -700 |

例如：

```
Buy 后 60 秒内 Sell
→ 触发 Quick Sell Curse
→ -300 分
```

---

## 12.8 Anti-Wash Penalty

防止刷交易。

| 行为 | 惩罚 |
| --- | --- |
| 30 秒内来回 Swap 超过 3 次 | -500 |
| 同一区块内反向 Swap | 本次动作不计分 |
| 快进快出 LP 少于 60 秒 | -200 |
| 同钱包重复刷同一隐藏事件 | 第二次起收益递减 80% |

# 14. 奖池分配规则

每个副本有自己的 Prize Pool。

奖池来源：

```
1. 创建者初始注入
2. Entry Fee 的 50%
3. Hook Fee 的 40%
4. Hint Fee 的 50%
```

---

## 14.1 副本排行榜奖励

每个副本赛季结束后，排行榜前 50 名瓜分奖池。

只有成功通关的玩家有资格参与分奖。

| 排名 | 奖池比例 |
| --- | --- |
| 第 1 名 | 25% |
| 第 2 名 | 15% |
| 第 3 名 | 10% |
| 第 4-10 名 | 25%，按分数占比分配 |
| 第 11-50 名 | 25%，按分数占比分配 |

如果通关人数少于 50 人：

```
未分出的奖励 70% 滚入下一期副本奖池
30% 按当前通关玩家分数占比追加分配
```

# 15. 创作者创建副本规则

创建者发布副本需要支付：

```
100 QUSD
```

这 100 QUSD 在 MVP 中固定拆分为：

| 去向 | 数值 |
| --- | --- |
| 平台创建费 | 20 QUSD |
| 最低初始奖池 | 80 QUSD |

说明：

```
创建 Agent 的最低成本 = 100 QUSD
其中 20 QUSD 作为平台创建费
其中 80 QUSD 注入该 Agent 的初始 Prize Pool
```

信誉保证金、额外奖池注入、创作者信誉机制可以作为后续版本扩展。

---

# 16. 创作者收益释放规则

创建者收入不是实时到账，而是进入 Escrow。

释放条件如下：

| 条件 | 释放比例 |
| --- | --- |
| 至少 1 名玩家通关 | 释放 20% |
| 至少 10 次有效挑战，且至少 3 名玩家通关 | 再释放 20% |
| 玩家评分 ≥ 4.0 / 5 | 再释放 20% |
| 争议率 < 5% | 再释放 20% |
| 赛季结束且副本规则无异常 | 释放剩余 20% |

如果副本长期无人通关：

```
50 次有效挑战后仍 0 人通关
或
上线 24 小时后仍 0 人通关
```

副本进入 Review 状态。

Review 期间：

```
1. 创建者收益冻结
2. 副本曝光降低
3. 平台检查隐藏通关路径
```

如果确认副本不可通关：

```
1. 创建者未释放收益退回玩家保护基金
2. 后续版本如启用信誉保证金，则保证金 70% 用于补偿玩家
3. 后续版本如启用信誉保证金，则保证金 30% 进入平台审核基金
4. 创建者信誉降级
```

# 17. 可通关性验证规则

创作者发布副本时，必须先提交 Agent 设定和玩法 Prompt。

系统会调用 AI 生成一条隐藏通关路径。

创作者可以在自己的创建/管理页面查看完整隐藏路径，并确认是否发布；玩家端永远不能看到真实路径。

例如：

```
Donate → Add LP → Buy → Hold 180 秒 → Sell
```

系统会自动模拟验证：

```
1. 路径是否可以完成
2. 需要的最小资金是否 ≤ 100 QUSD
3. 是否存在无法避免的惩罚
4. 15 分钟内是否能完成
5. 最终净值是否 ≥ 95 QUSD
```

验证不通过，不允许发布。

副本发布后，系统会保存：

```
Rule Hash
Solution Hash
```

玩家看不到完整解法，但可以知道副本规则已经锁定。

# 18. Agent 个性化提示规则

每个 Agent 必须有自己的世界观词库。

例如 Dragon Agent：

| 游戏动作 | 龙世界观表达 |
| --- | --- |
| Donate | 献上烟雾 |
| Add LP | 披上龙鳞 |
| Buy | 拿起金币 |
| Hold | 等火焰低头 |
